There are several other input types.

The checkbox input type allows us to have a checkbox with text next to it.

Or we can have it come pre-checked too.

And even choose the side the text/checkbox is on by putting the text between the label after the input element.

As long as the name, id and label names are synced, we only have to click on the text to check the boxes.

A button is needed to submit the form.

Radio input

This one is a selection that works on it´s own for each radio. To sync them we need to set the input name of each to the same thing.

Like this to select a size and only one option.

Except here we missed the very important value for the input, it is what returns the selection to us. If you click the button, the address bar does not show anything regarding the user´s selection, but the example below does! This one just says sizes=on.

Select

Select needs a name and id, which come ready for us to enter information. We also need to call that same name for the label for option to sync them together.

All the options must be inside the select elements.

We can have any of the option pre-selected by using selected on them.

What will show on the browser after submitting the form will be the value we assigned, not the text of the option. For example, the low carb option would show lowcarb instead of Low-carbohydrate.
